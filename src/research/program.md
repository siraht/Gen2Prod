# Autonomous research program

Work on one declared track at a time. Form one falsifiable hypothesis, change one bounded policy/pass/verifier field, run the frozen fixture split under the same accounting budget, and accept only if every hard evaluator mutation control still fires and the lexicographic fitness vector improves. Never edit `prepare.ts`, `evaluate.ts`, gate definitions, fixture outputs, or gold artifacts during an experiment. Rejected candidates remain useful preference and verifier data.

The policy track edits policy decisions only. The pass track changes one pass variant only. The verifier track must pass frozen positive/negative controls and may never accept by weakening a gate. Hidden holdout results are recorded periodically but are not used to search each candidate.
